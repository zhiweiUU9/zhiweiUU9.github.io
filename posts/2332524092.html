Linux下安装MySQL
MySQL安装

阅读时长:   4 分

安装前准备

1.检查是否已经安装过mysql，执行以下命令：

rpm -qa | grep mysql


如果存在，执行以下命令

rpm -e --nodeps mysql-libs-5.1.73-5.el6_6.x86_64

2. 查找所有mysql相关的文件夹

whereis mysql

find / -name mysql

如果存在，执行以下命令删除：

rm -rf 文件地址1 文件地址2  文件地址2

MySQL下载地址

MySQL官方下载地址：https://dev.mysql.com/downloads/mysql/

下载安装MySQL

1.下载并移动MySQL至/usr/local/目录下

# 下载mysql
wget https://dev.mysql.com/get/Downloads/MySQL-5.7/mysql-5.7.30-linux-glibc2.12-x86_64.tar.gz

# 下载完成后解压MySQL
tar -xzvf mysql-5.7.30-linux-glibc2.12-x86_64.tar.gz


# 将解压好的文件移动到/usr/local/目录下
mv mysql-5.7.30-linux-glibc2.12-x86_64 /usr/local/

cd /usr/local/

# 重命名为mysql
mv mysql-5.7.30-linux-glibc2.12-x86_64 mysql

2.在/usr/local/mysql目录下创建data目录

cd /usr/local/mysql

mkdir data

3.更改mysql目录下所有的目录及文件夹所属的用户组和用户，以及权限

chown -R mysql:mysql /usr/local/mysql

chmod -R 755 /usr/local/mysql

如果出现chown: 无效的用户: “mysql:mysql”的错误，需要先创建组和用户。

#创建mysql组
groupadd mysql

#创建mysql用户添加到mysql组
useradd -r -g mysql mysql

4.编译安装并初始化mysql,务必记住初始化输出日志末尾的密码（数据库管理员临时密码）

cd /usr/local/mysql/bin

./mysqld --initialize --user=mysql --datadir=/usr/local/mysql/data --basedir=/usr/local/mysql

5.运行初始化命令成功后，输出日志如下：

记住末尾那段字符串，此字符串为mysql管理员临时登陆密码。

6.编辑配置文件my.cnf，添加配置如下

vi /etc/my.cnf


[mysqld]
datadir=/usr/local/mysql/data
basedir = /usr/local/mysql

character-set-server=utf8

socket=/var/lib/mysql/mysql.sock
# Disabling symbolic-links is recommended to prevent assorted security risks
symbolic-links=0
# Settings user and group are ignored when systemd is used.
# If you need to run mysqld under a different user or group,
# customize your systemd unit file for mariadb according to the
# instructions in http://fedoraproject.org/wiki/Systemd

[mysqld_safe]
log-error=/usr/local/mysql/data/mariadb/mariadb.log
pid-file=/usr/local/mysql/data/mariadb/mariadb.pid

7.启动mysql服务器

/usr/local/mysql/support-files/mysql.server start

显示如下，则代表成功

如果出现如下错误，

出现错误原因说明:
1. 在这个目录下没有这个目录或文件，并且用户权限不够，无法自动创建。

2. 说白了也就是权限不够，无法创建socket套接字连接文件

解决方法：
7.1需要在/usr/local/mysql/data/创建目录mariadb以及文件mariadb.log

mkdir mariadb

cd mariadb

touch mariadb.log

7.2给/usr/local/mysql/下的data目录升权限

chmod -R 777 data

7.3给/var/lib下的mysql目录升权限

chmod -R 777 mysql

8.添加软连接，并重启mysql服务

ln -s /usr/local/mysql/support-files/mysql.server /etc/init.d/mysql

ln -s /usr/local/mysql/bin/mysql /usr/bin/mysql

service mysql restart

9.登录mysql，修改密码(密码为第5步的临时密码)

mysql -u root -p

输入完密码后，如果报以下错误：

是因为你my.cnf 配置文件中设置了 [mysqld] 的参数 socket ，而没有设置[client]的参数socket。

解决方法：
添加 [client] 配置项，如下所示

[client]
port=3306
socket=/var/lib/mysql/mysql.sock


然后重启服务

service mysql restart

再次重新输入密码后就成功了。

10.修改密码

set password for root@localhost = password('yourpass');

11.开放远程连接

use mysql;

update user set user.Host='%' where user.User='root';

flush privileges;

12.设置开机自动启动

1、将服务文件拷贝到init.d下，并重命名为mysql
cp /usr/local/mysql/support-files/mysql.server /etc/init.d/mysqld
2、赋予可执行权限
chmod +x /etc/init.d/mysqld
3、添加服务
chkconfig --add mysqld
4、显示服务列表
chkconfig --list

到这里就结束了，过程很坎坷，步步报错，还好都解决了。
